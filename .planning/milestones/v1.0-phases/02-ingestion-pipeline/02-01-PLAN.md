Create `bench/longmemeval/ingest.rs` with:

1. **Config types** (per CONTEXT.md locked decisions):
   - `IngestFormat` enum: `Text` (default), `Json`. Derive Serialize, Deserialize, Debug, Clone, Copy, PartialEq, Eq. `#[serde(rename_all = "kebab-case")]`.
   - `ConsolidationMode` enum: `End` (default), `PerSession`, `Off`. Same derives. Add `enabled()` and `per_session()` helper methods.
   - `IngestConfig` struct: `format: IngestFormat`, `consolidation: ConsolidationMode`. Derive Default (both fields default to their enum defaults).

2. **Result types**:
   - `IngestResult`: `question_id: String`, `bank_id: BankId`, `stage_metrics: BTreeMap<LlmStage, StageUsage>`, `stats: IngestStats`, `timing: IngestTiming`. Derive Serialize, Deserialize, Debug, Clone.
   - `IngestStats`: `sessions_ingested`, `facts_stored`, `entities_resolved`, `links_created`, `opinions_reinforced`, `opinions_weakened`, `session_failures`, `observations_created`, `observations_updated` (all usize). Derive Default, Serialize, Deserialize, Debug, Clone.
   - `IngestTiming`: `ingest_time_s`, `consolidation_time_s`, `total_time_s` (all f64). Derive Default, Serialize, Deserialize, Debug, Clone.

3. **Date parsing** (LongMemEval format: `"2023/05/20 (Sat) 02:21"`):
   - `pub fn parse_date_prefix(date_str: &str) -> String` — extracts `"[Date: 2023-05-20]"`. Take everything before ` (`, replace `/` with `-`, wrap in `[Date: ...]`.
   - `pub fn parse_haystack_date(date_str: &str) -> DateTime<Utc>` — parse to `DateTime<Utc>`. Strip the `(Day)` portion by splitting on `(` and `)` to get `"2023/05/20 "` and `" 02:21"`, join and parse with `NaiveDateTime::parse_from_str("%Y/%m/%d %H:%M")`. Fallback: try date-only with `NaiveDate::parse_from_str("%Y/%m/%d")`. Last resort: `Utc::now()`.

4. **Session formatters** (per CONTEXT.md locked decisions):
   - `pub fn format_session_text(turns: &[Turn], date_str: &str) -> String` — produces `[Date: YYYY-MM-DD]\n\nuser: content\nassistant: content`. Uses `parse_date_prefix` for the date line.
   - `pub fn format_session_json(turns: &[Turn]) -> String` — serializes turns as a JSON array of `{"role": "...", "content": "..."}` objects. Since `Turn` only has `role` and `content`, serialize the structs directly with `serde_json::to_string`. No `has_answer` leakage.

   Note: `format_session_json` does NOT take `session_id` or `date_str` parameters. The date prefix is embedded in the content via `parse_date_prefix` in the caller (ingest loop), not inside the JSON formatter. For raw JSON mode, the date prefix is prepended as `"[Date: ...]\n\n" + json_string` in the ingest loop, matching the same pattern as text mode.

5. **Module re-export**: Update `bench/longmemeval/mod.rs` to add `pub mod ingest;`.

6. **Unit tests** in `#[cfg(test)] mod tests`: Cover all behaviors listed above. Use `make_turns()` helper returning a small vec of Turn values.

  <action>
Add the main `ingest_instance` function to `bench/longmemeval/ingest.rs`:

```rust
pub async fn ingest_instance(
    instance: &LongMemEvalInstance,
    runtime: &ElephantRuntime,
    config: &IngestConfig,
) -> Result<IngestResult>
```

Implementation (follow the skeleton from 02-RESEARCH.md Pattern 4/5/6):

1. **Create bank** (INGEST-01):
   - `MemoryBank { id: BankId::new(), name: format!("longmemeval-{}", instance.question_id), mission: "Long-term conversational memory benchmark".into(), directives: vec![], disposition: Disposition::default(), embedding_model: runtime.embeddings.model_name().to_string(), embedding_dimensions: runtime.embeddings.dimensions() as u16 }`
   - Call `runtime.store.create_bank(&bank).await?`

2. **Sequential session loop** (INGEST-02, INGEST-04):
   - Iterate `instance.haystack_sessions.iter().zip(instance.haystack_dates.iter()).enumerate()`
   - For each session: format content based on `config.format` (Text or Json). For BOTH modes, the date prefix `parse_date_prefix(date_str)` goes at the top, then the formatted content. Text mode: `format_session_text(session, date_str)` (already includes prefix). Json mode: `format!("{}\n\n{}", parse_date_prefix(date_str), format_session_json(session))`.
   - Parse timestamp: `parse_haystack_date(date_str)`
   - Call `runtime.retain.retain(&RetainInput { bank_id: bank.id, content, timestamp, turn_id: None, context: None, custom_instructions: None, speaker: None }).await`
   - On Ok: accumulate stats (facts_stored, entities_resolved, links_created, opinions_reinforced, opinions_weakened, sessions_ingested += 1)
   - On Err: log with `tracing::warn!`, increment `stats.session_failures += 1`, continue to next session
   - Per-session consolidation (INGEST-03): if `config.consolidation.per_session()`, call `runtime.consolidator.consolidate(bank.id).await` after each session. On Ok: accumulate observations_created/updated. On Err: log warning.

3. **End-of-ingestion consolidation** (INGEST-03):
   - If `config.consolidation.enabled() && !config.consolidation.per_session()`: call `runtime.consolidator.consolidate(bank.id).await`. Track timing separately. Accumulate observations_created/updated.

4. **Return IngestResult**:
   - `question_id: instance.question_id.clone()`
   - `bank_id: bank.id`
   - `stage_metrics: BTreeMap::new()` (metrics collector integration is optional for now -- the scoped collector pattern from LoCoMo can be wired in Phase 3 when the CLI builds the runtime)
   - `stats` and `timing` from accumulated values

5. **INGEST-05 (pool sizing)**: NOT addressed in this phase per user decision. Add a doc comment on `ingest_instance`: `// Note: INGEST-05 (pool sizing for concurrent ops) is deferred to Phase 5. This function assumes sequential invocation.`

Add progress logging: use `tracing::info!` at start of ingestion (question_id, session count), and `tracing::debug!` per-session (session index/total).
