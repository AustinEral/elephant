<task type="auto">
  <name>Task 1: Create longmemeval-view binary with single-file and comparison modes</name>
  <files>bench/longmemeval/view.rs, Cargo.toml</files>
  <action>
1. Add `[[bin]]` target to Cargo.toml:
   ```toml
   [[bin]]
   name = "longmemeval-view"
   path = "bench/longmemeval/view.rs"
   ```

2. Create `bench/longmemeval/view.rs` with the following structure. Use `#[path = "../common/mod.rs"] mod common;` for shared imports (same pattern as longmemeval.rs). Import `common::io::sidecar_path`.

3. Define view-local deserialization types with `#[serde(default)]` on ALL fields (reader resilience, not writer types). Mirror the BenchmarkOutput schema from longmemeval.rs but with defaults everywhere:
   - `ViewBenchmarkOutput` (top-level)
   - `ViewManifest`
   - `ViewCategoryResult` { accuracy: f64, count: usize }
   - `ViewQuestionResult` (for JSONL sidecar)
   - `ViewStageUsage` { input_tokens: u64, output_tokens: u64, requests: u64 }
   Note: For `stage_metrics`, deserialize as `BTreeMap<String, ViewStageUsage>` (String keys, not LlmStage enum -- avoids depending on elephant::metrics in the view binary and handles forward/backward compat).

4. CLI parsing: `longmemeval-view [--verbose] <file.json> [file2.json]`
   - 1 positional arg: single-file mode
   - 2 positional args: comparison mode
   - `--verbose` flag: show per-question table (loaded from JSONL sidecar)
   - `--help` flag: usage message

5. **Single-file mode** (default output):

   a. **Config table** (using tabled with ConfigRow):
   ```
   config           | value
   -----------------+------------------
   profile          | full-s
   dataset          | longmemeval_s...
   consolidation    | end
   instance_jobs    | 4
   retain_model     | claude-3.5-son...
   reflect_model    | claude-3.5-son...
   embedding_model  | text-embedding...
   reranker_model   | local
   judge_model      | gpt-4o
   tag              | my-run
   commit           | abc1234
   timestamp        | 2026-03-15T...
   ```
   Show key config fields. Omit fields that are empty/default.

   b. **Per-category accuracy table** (using tabled with CategoryRow):
   ```
   category                    | acc    | n
   ----------------------------+--------+----
   single-session-user         | 85.0%  | 40
   single-session-assistant    | 90.0%  | 30
   single-session-preference   | 75.0%  | 20
   multi-session               | 88.0%  | 50
   knowledge-update            | 82.0%  | 40
   temporal-reasoning          | 95.0%  | 20
   abstention                  | 70.0%  | 30
   ----------------------------+--------+----
   overall                     | 85.7%  | 230
   ```
   Sort categories alphabetically, overall row last. Format accuracy as `{:.1}%`.

   c. **Stage metrics table** (using tabled with StageRow):
   ```
   stage        | requests | input_tok | output_tok
   -------------+----------+-----------+-----------
   Retain       | 500      | 1200000   | 50000
   Consolidate  | 500      | 800000    | 30000
   Reflect      | 500      | 600000    | 25000
   Judge        | 500      | 200000    | 10000
   ```
   Show stages that have >0 requests. Sort alphabetically by stage name.

   d. **Total time** line: `Total time: {:.1}s`

6. **Comparison mode** (two files):

   a. Config table shows both files side-by-side where values differ:
   ```
   config           | A              | B
   -----------------+----------------+----------------
   tag              | baseline       | improved
   judge_model      | gpt-4o         | gpt-4o
   ```

   b. Per-category table with delta:
   ```
   category                    | A      | B      | delta  | n(A) | n(B)
   ----------------------------+--------+--------+--------+------+------
   multi-session               | 85.0%  | 90.0%  | +5.0%  | 50   | 50
   overall                     | 85.7%  | 88.3%  | +2.6%  | 230  | 230
   ```
   Delta computed from accuracies: `(b - a) * 100.0`. When counts differ, show both n(A) and n(B). Use `fmt_pct_delta` style formatting (copy the helper from bench/view.rs -- positive deltas shown with +).

   c. Stage metrics comparison with delta columns.

7. **Verbose mode** (`--verbose`):
   - Load the questions JSONL sidecar via `sidecar_path(&artifact_path, "questions")`
   - Parse each line as `ViewQuestionResult`
   - Display per-question table: `question_id | category | correct | status | elapsed_s`
   - In comparison mode with verbose, show questions side-by-side with correct/incorrect changes highlighted

8. Add unit tests:
   - `test_parse_args_single_file` -- verify single-file mode parsed
   - `test_parse_args_comparison` -- verify two-file mode parsed
   - `test_parse_args_verbose` -- verify --verbose flag
   - `test_format_category_rows` -- verify accuracy formatting and overall row
   - `test_format_category_rows_comparison` -- verify delta calculation
