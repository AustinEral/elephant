2. In `src/reflect/mod.rs`: In `reflect_inner()` at line 222, replace:
   ```rust
   let mut messages: Vec<Message> = vec![Message::text("user", query.question.clone())];
   ```
   with:
   ```rust
   let user_content = if let Some(ref tc) = query.temporal_context {
       format!("[Current date: {tc}]\n\n{}", query.question)
   } else {
       query.question.clone()
   };
   let mut messages: Vec<Message> = vec![Message::text("user", user_content)];
   ```
   Note: `tc` will already be the output of `parse_date_prefix()` which produces `[Date: YYYY-MM-DD]`. The reflect user message uses `[Current date: ...]` to distinguish from ingested session date prefixes. The `tc` value passed will be just the ISO date string (see step 3), so the final message will read `[Current date: 2023-05-25]`.

   Add a unit test `temporal_context_in_user_message` that constructs a `ReflectQuery` with `temporal_context: Some("2023-05-25".into())` and verifies the logic produces `"[Current date: 2023-05-25]\n\nWhat happened?"`. This is a pure logic test on the string formatting -- no async, no mocking needed.

3. In `bench/longmemeval/longmemeval.rs`: At the QA loop (line 1444), add `temporal_context` to the `ReflectQuery` construction:
   ```rust
   temporal_context: Some(ingest::parse_date_prefix(&instance.question_date)),
   ```
   Wait -- `parse_date_prefix` returns `[Date: 2023-05-25]` (with brackets). For the reflect message, we want just the date part `2023-05-25`. Instead, extract the date inline:
   ```rust
   temporal_context: Some({
       let d = instance.question_date.split(" (").next().unwrap_or(&instance.question_date);
       d.replace('/', "-")
   }),
   ```
   Or better: create a small helper `pub fn parse_date_iso(date_str: &str) -> String` in ingest.rs that returns just `"2023-05-25"` (the ISO date without brackets). This avoids duplicating the split logic. Then call `temporal_context: Some(ingest::parse_date_iso(&instance.question_date))`.

   Actually, simplest approach: just pass the raw `instance.question_date` string. The reflect_inner format will wrap it as `[Current date: 2023/05/25 (Thu) 14:30]` which is human-readable and gives the LLM the full datetime. This preserves time-of-day precision. Use:
   ```rust
   temporal_context: Some(instance.question_date.clone()),
   ```
   The reflect_inner code wraps it as `[Current date: {tc}]` which produces `[Current date: 2023/05/25 (Thu) 14:30]`. This is fine -- the LLM understands this format.
