## Unsatisfied Requirements

### DATA-06: `question_date` not forwarded to reflect agent

**Requirement:** "question_date parsed and passed to reflect agent as temporal context"
**Phase:** Phase 1 (data layer) / Phase 4 (wire to reflect)
**Status:** Partial — data parsed but wire missing

**Evidence:**
- Phase 1 correctly parses `question_date` as a String field on `LongMemEvalInstance`
- Phase 4 VERIFICATION marked Truth 1 ("Reflect agent receives question_date as temporal context") as VERIFIED
- However, integration check reveals `ReflectQuery` (src/types/pipeline.rs) has only 3 fields: `bank_id`, `question`, `budget_tokens` — no temporal context field
- `question_date` is never extracted from the instance in the QA loop and is not forwarded to reflect
- The reflect agent does not receive temporal context for any question

**Impact:** Temporal-reasoning and knowledge-update questions may score lower without temporal grounding. The reflect agent operates without knowing "when" the question is being asked.

### Integration Gap: `session_limit` CLI flag

- `--session-limit N` is parsed, stored in `RunConfig`, serialized to manifest, and printed in config summary
- But it is never applied: `IngestConfig` has no `session_limit` field; `ingest_instance` processes all sessions unconditionally
- Not tied to any named requirement — this is a debugging convenience flag that silently does nothing
